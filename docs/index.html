scriv-release

Changesets-style release automation on top of scriv. A GitHub Action that opens a "Changelog Preview" PR when fragments are pending, and tags a release once that PR is merged.

  • Source & README github.com/whitphx/scriv-release
  • Install the GitHub App One-click registration of a private App that mints the action's release token.
  • Quickstart End-to-end setup: scriv config, action wiring, author flow.
  • Token setup Why a GitHub App, manual fallback, PAT alternative.
  • PyPI pypi.org/project/scriv-release

Minimal usage

jobs:
  release:
    runs-on: ubuntu-latest
    permissions:
      contents: write
      pull-requests: write
    steps:
      - uses: actions/checkout@v6
        with:
          fetch-depth: 0
          persist-credentials: false
      - uses: whitphx/scriv-release@v0.3.0
        with:
          app-id: ${{ vars.RELEASE_APP_ID }}
          app-private-key: ${{ secrets.RELEASE_APP_KEY }}
